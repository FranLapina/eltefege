git rebase

El comando git rebase se utiliza para cambiar la base de una rama. Permite reorganizar el historial de commits, integrando los cambios de una rama en otra de forma lineal, en lugar de crear un commit de fusión como lo hace git merge.

⌨️ En terminal


    1. Asegúrate de haber guardado todo tu trabajo

    Si tienes cambios sin confirmar:

    git add .

    git commit -m "estos son mis cambios"

    2. Cambia a main y obtén los últimos cambios

    git checkout main

    git pull origin main

    Esto asegura que main esté actualizado.

    3. Vuelve a tu rama de sobre la que estabas trabajando

    git checkout mi_rama

    4. Rebasa sobre main

    git rebase main

    Qué sucede:

    • Git elimina temporalmente tus commits de la rama.
    • Mueve tu rama al último commit de main.
    • Reproduce tus commits uno por uno.

    5. Resuelve conflictos (si existen)

    Si hay conflictos, Git se detendrá e indicará qué archivos están afectados.

    Para cada archivo en conflicto:

    • Abrelo
    • Arregla el conflicto
    • Marca como resuelto:

    git add "archivo que quieras poner"

    Luego continúa:

    git rebase --continue

    Repite hasta que se complete el rebase.

    6. Realiza push con --force-with-lease

    Como rebase reescribe el historial, necesitas hacer push forzado:

    git push --force-with-lease origin mi_rama

    El --force-with-lease es más seguro que --force: asegura que no sobrescribas accidentalmente el trabajo de otra persona.

💻 En Visual Studio Code


    En la ventana de control de git dentro de VSCode, para realizar un rebase deberás abrir la terminal integrada y ejecutar los comandos git rebase que se han mencionado anteriormente, ya que git rebase no tiene interfaz gráfica directa en VSCode.

    Si durante el rebase hay conflictos, VSCode te mostrará visualmente los archivos con conflictos en la sección de "Source Control". Puedes hacer click en "Resolve in Merge Editor" para resolver los conflictos de forma interactiva. Una vez resueltos todos los conflictos, deberás hacer click en "Commit" o ejecutar git rebase --continue en la terminal para completar el rebase.
